Chapter 1: Create an OPE Account

 

  • Open the website: https://www.bsnbase.com. Select [Login] -> [Register] to start the application process.


  • Fill in the required details above and press [Confirm] to submit the application form. 
  • After registration, a link will be sent to the email address that you have provided. Please follow the link to set up the login password for your account.
  • Your account has been successfully activated. You can now use the username and password to log into the OPE system.
  • You are registered as an “Individual Account”. If you would require a commercial invoice, please apply as a “Corporate Account”: After logging into the portal, go to [User Center] -> [Corporate Authentication], and upload an identity authentication. 
  • Your application will be reviewed automatically. Once the account has been verified, a notification will be sent to the registered mobile phone and email address.





 

Chapter 2: Connect OPE

 

 

OPE (Open Permissioned Ethereum), known as the “Wuhan chain”, is an Ethereum blockchain operated by Neptune Digital Limited.



2.1 OPE Gateway Access Instructions

 

Cryptographic Algorithm

The Wuhan Chain (based on Ethereum) generates chain addresses and public and private keys through ECDSA (secp256k1) algorithm.

 

Chain ID

The Wuhan Chain ID is 5555

 

Connecting Method

Wuhan Chain (based on Ethereum) currently supports JSON-RPC API and WebSocket connections. API documentation reference: https://eth.wiki/json-rpc/API

 

Key Mode

Supports Key Escrow, Uploading Public Key and Registering Chain Address modes.

Under uploading public key mode and registering chain address mode, users can manage their own keys and perform transactions in accordance with the API documentation. 

Under key escrow mode, BSN signs the blocks on behalf of the user so the eth_sendTransaction and eth_sendRawTransaction interfaces will not be able to complete transactions by the message format in the API documentation. The data needs to be submitted in the following format:

Blockchain Explorer URL

Otherwise, users can set up their own Wuhan Chain explorer. Details are as follows:

 

  • Prerequisites

node v10.15.3 or later

npm v6.4.1 or later

 

  • Installation

https://github.com/xops/expedition

 

  • Clone or download the project, and install dependencies

git clone https://github.com/xops/expedition.git && cd expedition && npm install

 

  • Start the explorer

npm start

The explorer will run at http://localhost:3000/

 

  • Set rpc via url

 ?rpcUrl=[gateway address]/api/[projectId]/rpc

Example: 

http://localhost:3000/?rpcUrl=https://opbningxia.bsngate.com:18602/api/[projectId]/rpc

Instructions for naming [projectid]:

For key escrow mode, please add BsnAccount within the url. Not applicable to other modes.

 

  • Configure default urls via environment variables

 REACT_APP_ETH_RPC_URL=https://opbningxia.bsngate.com:18602/api/[projectId]/rpc npm start 

 

RPC url

[gateway address]/api/[projectid]/[BsnAccount]/[Protocol]

If you access using the project key, please compose the request address using the above format and add x-api-key:{ProjectKeyValue} in the http header.

 

Note: Under key escrow mode, BsnAccount must be added to indicate the signature from BSN. This is not required for other modes: users can set their default signature and the url will only be used during transactions.

Chapter 4 Manage Account

 

4.1 My Account

 

1. Top-up

 

Deductions due to service release, product listing and other procedures will be debited from my account. Users must top up to ensure that there is sufficient balance in the account.

 

Instructions to top up:

 

  • Go to [User Center] -> [My Account] -> [Top up]

 

 

  • Enter the value to top up and click [Top up] to continue the payment. Once the payment is completed, the top-up will be added automatically.

 

  • Note: If users choose to remit offline, please enter the remittance identification code to all the fields that you can leave a comment and transfer the correct amount to the designated account. The top-up will arrive in 1-3 working days. If it has not been received within 3 working days, please contact the customer service (telephone number: 400-071-8215, working days 08:00-17:30).

 

  • Top-up by offline remittance is not supported by individual accounts.
  • For corporate accounts, please ensure that you are using a business bank account for transactions. Otherwise, supporting documents must be provided and it may require a longer time to issue a commercial invoice.



2. Withdraw

 

Users can withdraw balance in the account. The instructions are as follows:

 

  • In [Personal Center] -> [My Account], click [Withdrawal] and follow the page 



  • Enter the withdrawal amount and select [Apply for withdrawal] to proceed. The balance will be deducted immediately and the application will be reviewed.
  • If successful, the balance will be transferred to the same account from which the top up is charged. Any balance generated due to coupons or other promotions cannot be withdrawn.



3. Refund

 

Monthly bills and data usage bills are not allowed to be refunded. For annual bills with 

the remaining account period exceeding 1 month, you get a refund for any remaining full 

months. If you need a refund, please contact customer service (telephone number: 400-071- 8215, weekdays 08:00 - 17:30).

Please refer to 4.3 My Invoice.

4.2 My Invoice

 

Display bills generated by application products, application services, IPFS services, open consortium chain service, cross-chain services and proprietary node services.

 

The fees incurred by application services mainly include resource fees and data traffic fees. A prepayment model is used for resource fees, which depends on the TPS, storage capacity and number of billing nodes; whereas the data traffic fees uses a post-payment model, which depends on the actual traffic usage.

 

The fees incurred by application products is mainly the product launching fee, which is settled on a prepayment basis.

 

The fees incurred by IPFS services mainly include resource usage fees and data traffic fees. The resource usage fees are settled by prepayment, which is calculated according to the capacity; the data traffic fees are settled by post-payment, which is calculated according to the actual traffic usage.

 

The costs for the open consortium chain are mainly to allocate energy value or to allocate resources, which is settled by prepayment.

 

The fees incurred by cross-chain services are charged according to the number of cross-chain requests from the initial chain, which is settled by post-payment.

 

The fees incurred by proprietary node services mainly include resource fees and data traffic fees.

 

The resource usage fees are settled by prepayment, which depends on the cloud platform, underlying framework, region, number of nodes, host configuration and disk capacity; the data traffic fees are settled on a post-payment basis, which depends on the actual traffic usage.

 

The costs incurred by DCC management are mainly energy value recharge fees, DDC generating fees and DCC sending fees. All of which are prepaid.

 

An invoice will be generated when the above charges are incurred, including:

 

  • Service release: payment is settled when the service application is released; the amount is calculated based on TPS, storage capacity, and accounting nodes;

 

  • Service configuration upgrade: paid upon configuration upgrade, for the difference between the new configuration and the old configuration in the remaining period of the billing cycle;

 

  • Service periodic deduction: after the service is released, the system deducts the payment periodically on a yearly or monthly basis;

 

  • Periodic deduction of traffic: after the service is released, the system will deduct the payment on a weekly basis according to the actual traffic usage;

 

  • Product launch: payment is settled when the product is applied for launch;

 

  • Product periodic deduction: after the product is launched, the system will deduct the payment on a monthly basis;

 

  • IPFS service activation: settled when the service is activated; the amount is calculated according to the capacity;

 

  • IPFS service upgrade: paid upon service upgrade, for the difference between the new configuration and the old configuration in the remaining period of the billing cycle;

 

  • IPFS service cycle: after the service is activated, the system periodically deducts payments on a yearly or monthly basis;

 

  • IPFS traffic cycle: the system deducts payments on a weekly basis according to the actual traffic usage of file downloads;

 

  • Open consortium chain deduction: after the chain account is activated, deductions will be made when energy or resources are allocated;

 

  • Cross-chain service cycle: the system deducts payments on a weekly basis according to the actual number of cross-chains;

 

  • Proprietary node service activation: payment is made when the service is activated, and the amount is calculated based on the cloud platform, underlying framework, region, number of nodes, host configuration and disk capacity;

 

  • Proprietary node service cycle: after the service is activated, the system will deduct money periodically on a yearly or monthly basis;

 

  • Proprietary node traffic cycle: the system deducts payments on a weekly basis according to the actual traffic usage;

 

  • Proprietary node configuration upgrade: paid upon service upgrade, for the difference between the new configuration and the old configuration in the remaining period of the billing cycle;

 

  • Energy value top-up: after a chain account is activated, deductions will be made when the energy value is recharged;

 

  • DCC generation: deduct when the DDC is generated

 

  • DDC sending: deduct when the DDC is sent



 

 

4.3 My Invoice

 

Users can apply for invoicing from the OPE. The steps are as follows:

 

  • Go to [User Center] -> [My Invoice] -> [Add Invoice];




  • The user selects the bill which needs an invoice. The billing amount will be calculated based on the actual value charged from the selected bill. Press [Next] to view the invoice details;



 

  • Select the invoice header and invoice recipient, and select [OK] to submit the invoicing application.
  • If the request is accepted, a paper and electronic invoice will be sent to the invoicing applicant address and email address.
